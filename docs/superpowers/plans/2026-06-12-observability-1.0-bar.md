# Observability 1.0 Bar: Dashboards, Alerts, Runbooks (#8 / compliance addendum)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the operability artifacts the compliance addendum names as the family maturity bar before 1.0: a Grafana dashboard, PrometheusRule alerts with runbooks, and a complete conditions/reason-code catalogue, for the metrics the system ALREADY exports. No new metric is invented; this slice makes the existing telemetry operable. Verified: the PrometheusRule expressions are validated by `promtool check rules` in CI (a real syntax/semantics gate), the dashboard JSON is valid and references only real metric names, each alert links a runbook that exists, and the conditions catalogue covers every condition type the controllers set. The artifacts are applyable (a deploy/monitoring kustomize layer, opt-in).

**Honesty constraint:** every metric referenced exists in the code (`internal/controller/metrics.go`: mitos_claim_pending_total, mitos_orphan_sweeps_total, mitos_claim_errors_total{pool,reason}, mitos_pool_ready_snapshots{pool}; `internal/daemon/server.go`: mitos_fork_duration_seconds histogram, mitos_active_sandboxes, mitos_memory_shared_bytes, mitos_memory_unique_bytes, mitos_cow_memory_savings_bytes, mitos_metered_disk_bytes). Alert THRESHOLDS are defensible, not fabricated SLO numbers: latency alerts fire on a clearly-labeled, environment-tunable threshold (the <=10ms claim is a bare-metal TARGET, not the alert threshold; the fork-duration alert fires on a generous shared-environment value and the runbook says to tune it per cluster). Rate/ratio alerts (error rate, pending sustained, warm-pool starvation, orphan-sweep spike) use relative/for-duration conditions, not magic absolute numbers, and the runbook explains the signal. The dashboard panels are labeled with what they actually measure (the CoW panels show the shared-once dedup, honestly, not a raw figure). No alert or panel claims an SLO a benchmark has not established; where a number is a target it is labeled a target.
