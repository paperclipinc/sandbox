# CoW-Aware Metering Implementation Plan (issue #33)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close issue #33, a format-freeze blocker. Make memory and disk metering CoW-aware so the reported node footprint reflects the physical reality of copy-on-write forking, not a naive per-sandbox sum that double-counts the shared template. Today `GetCapacity` does `totalShared += s.MemoryShared` across every sandbox (`engine.go:913`), so N forks of one template report N times the template's shared memory as used, even though those forks map the same physical pages via `mmap(MAP_PRIVATE)`. That overcounts the footprint and, perversely, hides the system's core value (high density from CoW sharing). This PR counts the shared region once per distinct template, sums only the genuinely unique (private/dirty) pages per fork, and exposes an honest per-sandbox, per-template, and per-node metering report. The same correction is applied to CoW disk (reflink Snapshot volumes share blocks). The result is a metering surface a hosted service can bill on and an honest density number that satisfies the no-unverified-claims principle.

**Honesty constraint:** the reported "CoW-aware used" memory is `sum(unique per sandbox) + sum(shared once per distinct template)`. Per-fork `MemoryShared` from smaps approximates the template's shared resident set; we count it once per template (the representative value, documented), not per fork. We also keep and expose the naive sum so the CoW saving is visible and auditable, and we state precisely what is exact (per-process Private_Dirty from smaps_rollup) versus approximate (the shared-once attribution, since two forks of one template share the same physical pages but smaps reports each process's view). No density or savings number is printed that a test does not derive from real smaps in KVM CI.

**Architecture:** Each sandbox already records `MemoryUnique`/`MemoryShared` from `/proc/<pid>/smaps_rollup` (`readMemoryStats`) and knows its source template (the snapshot/template id it forked from). A new `internal/metering` package aggregates a snapshot of sandboxes into a `Report`: per-sandbox unique+shared+template, per-template shared-counted-once + fork count, and a node total `UsedCoWAware = sum(unique) + sum_over_templates(shared_once)` alongside `UsedNaive = sum(unique)+sum(shared)` and `CoWSavings = UsedNaive - UsedCoWAware`. `GetCapacity` uses the CoW-aware total for `MemoryUsed`. Disk: reflink Snapshot volumes that share a template seed are counted once per template volume; Fresh volumes count their own size. A forkd metering endpoint and corrected metrics expose the report.

**Context for the implementer:**
- `internal/fork/engine.go`: the `Sandbox`/sandbox record carries `MemoryUnique`, `MemoryShared`, the source template/snapshot id (find the field, e.g. `SnapshotID`), and `Pid`. `readMemoryStats(pid)` (engine.go:1183) reads smaps_rollup (Private_Clean/Private_Dirty -> unique; the shared accounting). `GetCapacity` (engine.go:906) sums naively today; `Capacity{MemoryUsed, MemoryShared}` is reported to the controller via heartbeat. The mock engine (mock.go) has fixed shared=256MB/unique=265KB values; keep the mock honest about CoW (it should also count shared once per template so mock-based tests of the aggregation pass).
- Daemon metrics: `internal/daemon/server.go` has `memoryShared`/`memoryUnique` gauges; point them at the CoW-aware values.
- Volumes: `internal/volume` Snapshot policy reflink-copies the template seed; multiple forks of a template share the seed's blocks until written. The template volume seed path is `<dataDir>/templates/<id>/volumes/<name>.ext4`; per-fork backings are `<dataDir>/sandboxes/<id>/volumes/<name>.ext4`.

### Task 1: `internal/metering` CoW-aware aggregation + fix GetCapacity

**Files:** Create `internal/metering/report.go` + `_test.go`; modify `internal/fork/engine.go` (GetCapacity), `internal/fork/mock.go`.

- [ ] `report.go`: input is a slice of per-sandbox samples `Sample{ID string; Template string; MemoryUnique, MemoryShared int64}` (a small struct the engine fills from its sandboxes; do not import the engine to avoid a cycle). `func Aggregate(samples []Sample) Report` where `Report{ Sandboxes []SandboxMetering; Templates []TemplateMetering; UsedCoWAware, UsedNaive, CoWSavings, TotalUnique int64 }`. Rules: `TotalUnique = sum(MemoryUnique)`; for each distinct Template, `shared_once = a representative MemoryShared for that template` (use the max of its forks' MemoryShared, documented as the conservative representative, since all forks of a template should report ~the same shared set); `UsedCoWAware = TotalUnique + sum_over_templates(shared_once)`; `UsedNaive = TotalUnique + sum(MemoryShared)`; `CoWSavings = UsedNaive - UsedCoWAware`. Per-template: fork count + shared_once. Handle empty input (all zeros), a single template, and multiple templates.
- [ ] `engine.go` GetCapacity: build `[]metering.Sample` from `e.sandboxes` (each sandbox's id, its source template id, MemoryUnique, MemoryShared), call `metering.Aggregate`, and set `Capacity.MemoryUsed = report.UsedCoWAware` and `Capacity.MemoryShared = sum_over_templates(shared_once)` (the honest shared footprint). Keep the existing fields' meaning documented.
- [ ] `mock.go`: ensure the mock's GetCapacity also routes through metering.Aggregate so mock-engine tests see CoW-aware totals (10 mock forks of one template -> shared counted once). Keep the per-fork mock values.
- [ ] Tests (metering/report_test.go): 10 samples from template "A" (each shared=256MB, unique=1MB) -> UsedCoWAware = 10MB + 256MB = 266MB (NOT 10*256+10 = 2570MB); UsedNaive = 2570MB; CoWSavings = 2304MB; per-template A has forkCount=10, shared_once=256MB. Two templates A and B -> two shared regions counted. Empty -> zeros. Single fork -> CoWAware == Naive. Also a fork/engine or mock test asserting GetCapacity.MemoryUsed is the CoW-aware value for multiple forks of one template.
- [ ] Commit `feat: CoW-aware memory metering counts shared template memory once`.
